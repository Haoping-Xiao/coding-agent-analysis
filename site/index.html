一个 Coding Agent
到底是怎么造出来的？

它不是魔法。剥开外壳，里面是一个朴素的循环：大模型动脑 → 调用工具动手 → 看结果 → 再来一轮。 本页用 5 个知名开源项目的真实代码，带你看清这套机制，以及每家偷偷做的「巧妙设计」。

第 0 课

先说人话：coding agent 是什么

普通聊天机器人只会「说」。Coding agent 会「干活」——它能读你的代码、改文件、跑命令、看报错，然后自己决定下一步。 差别就在于它手里有一套工具，以及一个让它反复行动的循环。

🧠

大脑：大模型

负责思考和决策。它读到当前对话和任务，输出「我要调用某个工具，参数是这些」。它本身不会真的碰你的电脑。

🛠️

双手：工具

读文件、写文件、跑 shell、搜索代码……这些「动手」的能力由程序提供，模型只是开口要求，由 agent 框架真正执行。

🔁

心跳：循环

执行完工具，把结果再喂回模型，让它接着想下一步。如此往复，直到任务完成。这就是 agent 区别于聊天机器人的关键。

第 1 课 · 共性

所有 coding agent 共享的「Agent Loop」

无论 Codex 用 Rust、Claude Code 用 TypeScript，骨架几乎一模一样。下面这张图是它们共同的心跳。 点亮每一步看它在干什么。

  1. 1

    接收输入

    用户的需求、或上一轮工具的结果

  2. 2

    模型思考

    组装上下文，请求大模型（流式返回）

  3. 3

    要求调用工具

    模型输出 tool_call：读文件 / 跑命令…

  4. 4

    审批 + 执行

    危险操作先问你，再在沙箱里执行

  5. 5

    观察结果 → 回到第 1 步

    把结果喂回模型，继续，直到没有新的工具调用

LLM
真实印证： Claude Code 的主循环就是一个 while (true) 的 queryLoop()（restored-src/src/query.ts）； Gemini CLI 把一轮模型往返封装成 Turn.run（packages/core/src/core/turn.ts）； Kimi Code 则是 runTurn → executeLoopStep，每个 step 都是「调模型，若 stop 原因是 tool_use 就执行工具」（packages/agent-core/src/loop/）。

第 5 课

30 行，写一个你自己的 mini agent

把前面讲的循环翻译成伪代码，你会发现真没那么神秘。

mini_agent.ts 
// 一个最小可用的 agent loop（去掉了流式/错误处理）
async function runAgent(task) {
  let messages = [{ role: "user", content: task }];

  while (true) {
    // 1) 模型动脑：给它对话历史 + 可用工具清单
    const reply = await llm.chat({ messages, tools: toolSchemas() });
    messages.push(reply);

    // 2) 没有工具调用 → 任务结束
    if (!reply.toolCalls?.length) return reply.text;

    // 3) 逐个执行工具（危险操作先审批）
    for (const call of reply.toolCalls) {
      if (needsApproval(call) && !(await askUser(call))) {
        messages.push(toolResult(call, "用户拒绝了该操作"));
        continue;
      }
      const output = await tools[call.name](call.args); // 读文件/跑命令…
      // 4) 观察：把结果喂回去，下一轮 while 继续
      messages.push(toolResult(call, output));
    }
  }
}

想更像真货？加这些

  • 流式输出 + 边流边执行只读工具（Claude Code 的并行思路）
  • 上下文满了自动压缩（compaction）
  • 沙箱执行 + 审批缓存（Codex 的思路）

想做多端？

  • 把核心做成 server，CLI/网页/IDE 当客户端（OpenCode）
  • 用类型化协议（JSON-RPC / OpenAPI）暴露能力（Codex / OpenCode）

想可调试 / 可恢复？

  • 把每个事件写进 wire log，崩溃后 replay 恢复（Kimi Code）
  • 改文件前打 git checkpoint，支持一键回滚（Gemini CLI）
